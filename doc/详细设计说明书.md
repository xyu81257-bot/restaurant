美味餐厅点餐管理系统详细设计说明书
1 概述
基于概要设计，细化后端分层类结构、核心业务流程、接口明细、页面交互逻辑，提供编码开发直接依据。
2 后端分层详细设计
2.1 分层职责
Controller 层：接收前端请求、参数校验、调用 Service、统一返回 Result 封装结果
Service 层：核心业务逻辑、事务控制、多模块联动调用
Mapper 层：数据库 CRUD 操作，MyBatis-Plus 封装基础方法
Entity 实体类：映射数据表字段
DTO 传输对象：前后端数据传输封装（LoginDTO、OrderDTO 等）
2.2 UML 类图说明
包含全部实体类、DTO、Controller、Service、ServiceImpl、Mapper，各业务服务依赖关系清晰；OrderServiceImpl 为核心业务类，依赖菜品、库存、桌台、订单明细服务，实现下单联动逻辑。
3 核心业务流程详细设计
3.1 下单完整流程
顾客提交购物车菜品 + 桌台 ID
后端循环校验菜品是否存在、是否上架、库存是否充足
校验通过生成唯一订单号，计算总金额
新增订单主表、订单明细记录
更新桌台状态为已占用
逐笔扣减对应菜品库存
清空用户购物车，返回订单信息
3.2 库存扣减流程
根据菜品 ID 查询库存记录
判断库存是否存在、库存数量是否大于购买数量
充足则扣减库存，更新更新时间；不足返回扣减失败
全程事务，失败回滚
3.3 支付流程
查询订单，校验订单状态为待支付、未支付
修改订单支付状态、订单状态，记录支付时间
事务保证状态更新原子性
3.4 管理员清台流程
将桌台状态更新为空闲
查询该桌台所有未完成订单
全部订单标记为已完成，未支付订单自动补支付记录
操作日志记录清台行为

统一约束见概要设计，包含用户、菜品、分类、桌台、库存、订单、统计全部接口，标注请求方式、路径、入参、返回数据、功能说明。
5 前端页面详细设计
顾客端页面
首页：轮播横幅、推荐菜品展示
点餐页：分类标签、菜品卡片、加入购物车按钮
购物车页：菜品数量修改、删除、结算下单
我的订单页：多状态筛选、订单详情查看、支付、取消订单
管理端页面
首页统计看板：营收卡片、热销排行榜、营业额折线图
菜品 / 分类 / 桌台 / 库存 / 订单管理表格页面，支持新增、编辑、删除、分页查询
6 异常处理详细设计
全局异常处理器捕获后端所有异常，统一返回 500 错误信息
业务自定义异常：菜品不存在、库存不足、订单状态异常等，返回精准提示文字
前端捕获 Axios 请求异常，弹窗提示用户
7 权限详细设计
登录后存储 token 至 localStorage
前端路由守卫：未登录跳转登录页，顾客禁止访问管理路由
后端拦截器校验 token 合法性，解析用户角色，拦截越权接口请求
