美味餐厅点餐管理系统概要设计说明书
1 引言
1.1 目的
完成系统顶层架构、技术选型、功能模块划分、数据库总体设计，明确各层级职责，为详细设计、编码开发提供顶层规范。
1.2 设计范围
整体架构、前后端技术选型、功能模块拆分、数据库总体 E-R 模型、全局接口规范。
2 系统总体架构设计
采用前后端分离三层 B/S 架构
前端表现层：Vue3 页面，分为顾客端、管理后台、ECharts 可视化模块，Axios 发起 HTTP 请求
后端业务层：Spring Boot 分层架构（Controller→Service→Mapper），处理业务逻辑、权限校验、事务控制
数据持久层：MySQL 数据库，MyBatis-Plus 实现对象与数据表映射
整体交互流程
前端 Axios 携带 JWT Token 请求后端 Controller → Controller 接收参数转发 Service → Service 处理业务逻辑调用 Mapper → Mapper 操作数据库；数据原路逐层返回前端渲染页面。
3 技术选型总体说明
后端技术栈
Spring Boot 2.7、MyBatis-Plus、MySQL8.0、JWT、Maven、Lombok、事务管理 @Transactional
前端技术栈
Vue3、Vue Router、Pinia、Element Plus、Axios、ECharts
4 系统功能模块划分
模块 1：用户认证模块
注册、登录、退出、获取当前登录用户、权限校验
模块 2：顾客点餐模块
菜品浏览、购物车、订单创建、支付、订单查询、取消订单
模块 3：菜品分类模块
分类新增、编辑、删除、查询
模块 4：菜品管理模块
菜品 CRUD、上下架、库存关联
模块 5：桌台管理模块
桌台 CRUD、状态更新、清台自动完结订单
模块 6：库存管理模块
库存增减、库存预警、库存扣减校验
模块 7：订单管理模块
订单全量查询、状态修改、订单明细
模块 8：数据统计模块
营收统计、热销菜品、营业额趋势图表
5 数据库总体设计
实体：用户、菜品分类、菜品、桌台、库存、订单、订单明细
实体关系：一对多、一对一关联
用户 1→N 订单；桌台 1→N 订单；分类 1→N 菜品；菜品 1→1 库存；订单 1→N 订单明细
统一规范：所有表增加 deleted 逻辑删除字段，主键自增 bigint 类型
6 全局接口规范
基础路径：/api 统一前缀（基础业务接口）
请求方式：GET 查询、POST 新增、PUT 修改、DELETE 删除
鉴权：请求头 Bearer Token 传递 JWT
统一返回 JSON 格式：code 状态码、msg 提示、data 数据体
通用状态码：200 成功，500 服务异常
7 部署概要
本地开发：前后端分开启动；生产部署可打包后端 jar 包 + 前端静态资源部署至 Nginx。
8 安全总体设计
密码 MD5 加密存储
JWT 无状态登录认证
后端接口权限拦截，区分顾客 / 管理员角色
业务事务保证数据一致性
