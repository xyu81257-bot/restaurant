美味餐厅点餐管理系统数据库设计说明书
1 数据库环境
数据库：MySQL 8.0
字符集：utf8mb4
存储引擎：InnoDB（支持事务、外键关联）
2 E-R 实体关系图
实体：用户、菜品分类、菜品、桌台、库存、订单、订单明细
实体关联关系：
菜品分类 (1) —— 菜品 (N)：一对多
菜品 (1) —— 库存 (1)：一对一
用户 (1) —— 订单 (N)：一对多
桌台 (1) —— 订单 (N)：一对多
订单 (1) —— 订单明细 (N)：一对多
菜品 (1) —— 订单明细 (N)：一对多
3 七张数据表详细结构
3.1 category 菜品分类表
id bigint 主键自增；name varchar (50) 非空；deleted int 默认 0（逻辑删除）
3.2 desk 桌台表
id bigint 主键；number varchar (20) 唯一桌台号；status varchar (20) 默认空闲；deleted 逻辑删除
3.3 dish 菜品表
id 主键；dish_name 菜品名；category_id 关联分类；price decimal 价格；stock 库存冗余；status 1 上架 0 下架；deleted
3.4 inventory 库存表
id 主键；dish_id 唯一关联菜品；quantity 库存数量；warning_line 预警线；update_time 自动更新；deleted
3.5 user 用户表
id 主键；username 唯一用户名；password MD5 加密 64 位；role 角色 admin/customer；deleted
3.6 orders 订单主表
id 主键；order_no 唯一订单号；user_id 下单用户；desk_id 桌台；total_amount 总金额；order_status 待支付 / 已支付 / 已完成 / 已取消；pay_status 未支付 / 已支付；create_time 创建时间；pay_time 支付时间；deleted
3.7 order_detail 订单明细表
id 主键；order_id 关联订单；dish_id 菜品；quantity 购买数量；price 下单快照单价；amount 小计；deleted
4 字段统一约束规范
主键统一 bigint 自增，避免 int 溢出
金额 decimal (10,2)，保留两位小数
所有表增加 deleted 逻辑删除，不物理删除数据
唯一字段建立唯一索引（用户名、桌台号、订单号）
外键关联字段建立普通索引，提升查询速度
5 数据库事务设计
下单、库存扣减、清台、支付等多表联动操作添加事务注解，出现异常全部回滚，保证数据一致性。
6 初始化 SQL 说明
sql 目录下 restaurant.sql 包含建表语句、基础测试数据（菜品、分类、桌台、管理员账号），可直接执行初始化数据库。
7 数据安全设计
用户密码加密存储，数据库无明文密码
敏感订单数据仅管理员可查看，前端做权限隐藏
不存储顾客手机号、身份证等隐私信息，仅保留账号基础信息
