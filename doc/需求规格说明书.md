美味餐厅点餐管理系统需求规格说明书
1 概述
1.1 系统目标
实现顾客自助线上点餐、餐厅后台全流程管理、经营数据可视化统计，替代传统纸质点餐，降低门店人力成本，提升用餐效率，同时完成软件工程课程全流程实践。
1.2 角色定义
顾客：普通到店消费用户，可注册登录、点餐、支付、查看订单
管理员：餐厅运营人员，后台管理菜品、桌台、库存、订单、查看营收数据
2 功能性需求
2.1 顾客端功能需求
用户注册：填写用户名、密码，自动分配顾客角色
用户登录：账号密码校验，颁发 JWT 令牌，身份鉴权
菜品浏览：按分类筛选菜品，展示菜品名称、价格、图片、上架状态
购物车管理：添加菜品、修改数量、删除菜品、清空购物车
下单创建订单：选择空闲桌台，系统自动校验菜品上架状态、库存余量，库存不足拦截下单
订单支付：修改订单支付状态，记录支付时间
订单管理：查看全部历史订单，按状态筛选；未支付订单可取消，取消后释放桌台、返还库存
2.2 管理端功能需求
菜品分类管理：新增、编辑、删除菜品分类
菜品管理：新增菜品、修改信息、上下架、删除菜品；菜品下架自动清理用户购物车
桌台管理：新增桌台、可视化展示桌台空闲 / 占用状态；结账清台自动完结桌台所有订单并释放桌台
库存管理：绑定菜品库存、设置库存预警线、库存不足提醒、增减库存
订单管理：查看全量订单、修改订单状态、查看订单明细
用户管理：查看系统所有账号、管理员账号维护
数据统计：今日营收、月度营收、订单总数、热销菜品 TOP5、营业额趋势折线图
2.3 核心业务约束
下单时库存不足禁止创建订单；
订单创建同步扣减库存、占用桌台；
所有订单、库存、桌台操作使用事务，保证数据一致性；
普通顾客无法访问管理后台接口，后端权限拦截。
3 非功能性需求
3.1 性能需求
页面加载响应≤3s，下单核心接口响应≤5s
支持 50 人同时在线无卡顿
金额计算保留两位小数，精度无误差
3.2 安全需求
用户密码 MD5 不可逆加密存储，禁止明文保存
JWT 令牌鉴权，请求头携带 token，无 token 禁止访问接口
前后端双重参数校验，防止非法输入
路由守卫拦截未登录、越权访问页面
3.3 可靠性需求
订单创建、库存扣减、桌台占用使用事务，失败自动回滚
采用逻辑删除，不物理删除历史订单、菜品数据，便于数据统计
3.4 易用性需求
顾客点餐流程 4 步以内完成，无复杂操作
所有操作弹窗提示成功 / 失败信息，页面加载状态提示
3.5 可维护性需求
前后端分层模块化开发，低耦合，便于迭代新增功能
代码统一注释规范，接口文档完整
4 数据需求
系统 7 张核心数据表：用户表、菜品分类表、菜品表、桌台表、库存表、订单表、订单明细表，详细字段约束见《数据库设计说明书》。
